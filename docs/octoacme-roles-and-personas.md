# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

---

## UX Designer

### Role Summary
UX Designers conduct user research, create intuitive interface designs, and validate usability across all touchpoints. They partner with Product Managers to understand user needs and with Developers to ensure designs are implementable and accessible.

### Responsibilities
- Conduct user research and usability testing
- Create wireframes, prototypes, and design systems
- Define interaction patterns and user flows
- Collaborate with Developers on feasibility and technical constraints
- Ensure designs meet accessibility and usability standards
- Participate in design reviews and iterate based on feedback

### Goals
- Deliver user-centered, intuitive solutions
- Reduce support burden through good design
- Improve user satisfaction and product adoption

### Typical Communication
- Design specs and prototypes shared with Developers and Product Managers
- Design reviews and user testing sessions
- Collaboration on accessibility and usability requirements
- Feedback loops during development

### Interaction Points
- **With Product Managers:** Align on user needs, success metrics, and feature priorities
- **With Developers:** Discuss technical feasibility, implementation timelines, and design system maintenance
- **With QA:** Define usability test cases and acceptance criteria for UI/UX

---

## DevOps Engineer

### Role Summary
DevOps Engineers design, build, and maintain infrastructure and deployment pipelines. They enable teams to release reliably and monitor production systems, ensuring high availability and performance.

### Responsibilities
- Design and maintain CI/CD pipelines
- Implement Infrastructure as Code (IaC) for repeatable, scalable deployments
- Manage deployment automation and rollback procedures
- Set up monitoring, alerting, and observability tools
- Collaborate with Developers on deployment requirements and best practices
- Manage production incidents and post-mortems
- Document runbooks and deployment procedures

### Goals
- Enable fast, reliable, and safe deployments
- Minimize downtime and production incidents
- Reduce deployment friction and manual work

### Typical Communication
- Runbooks and deployment documentation
- Incident response coordination
- Infrastructure and pipeline design discussions
- Regular on-call rotations and escalations

### Interaction Points
- **With Developers:** Support deployment and release processes, troubleshoot production issues
- **With Project Managers:** Coordinate deployment windows and release timelines
- **With Security/Compliance teams:** Implement security controls and compliance requirements in infrastructure

---

## Technical Writer

### Role Summary
Technical Writers create and maintain clear, accurate documentation for both internal teams and end users. They translate complex technical information into accessible guides, reducing support burden and improving product usability.

### Responsibilities
- Write and maintain user guides, API documentation, and tutorials
- Create and update release notes and change logs
- Develop onboarding and training materials for internal teams
- Collaborate with Developers to understand technical implementation
- Review documentation for accuracy, completeness, and clarity
- Manage documentation structure, versioning, and searchability
- Ensure consistency in tone, terminology, and formatting

### Goals
- Enable users to self-serve and reduce support tickets
- Accelerate team onboarding and knowledge sharing
- Maintain single source of truth for product documentation

### Typical Communication
- Documentation in wikis, help centers, and README files
- Collaboration with Developers on technical specs
- Feedback from Support and Product teams on documentation gaps
- Regular documentation audits and updates

### Interaction Points
- **With Developers:** Understand technical details, architecture, and APIs
- **With Product Managers:** Clarify feature requirements and success criteria
- **With Support/QA teams:** Identify documentation gaps and improvements needed

---

## Security Lead

### Role Summary
Security Leads identify, assess, and mitigate security risks throughout the project lifecycle. They champion security best practices, ensure compliance with policies, and work cross-functionally to build secure, trustworthy solutions.

### Responsibilities
- Identify and assess security risks and threats
- Conduct threat modeling and security reviews
- Define and enforce security requirements and standards
- Oversee security testing and vulnerability scanning
- Manage security incidents and coordinate incident response
- Ensure compliance with regulatory and organizational policies
- Mentor teams on security best practices and secure coding
- Maintain security documentation and runbooks

### Goals
- Minimize security vulnerabilities and breaches
- Build trust through secure, privacy-respecting solutions
- Enable teams to ship secure code confidently

### Typical Communication
- Security reviews and threat models
- Vulnerability reports and remediation plans
- Security incident notifications and post-mortems
- Regular security training and awareness sessions

### Interaction Points
- **With Developers:** Review code for security, provide secure coding guidance
- **With Product Managers:** Define security requirements and assess risk-benefit trade-offs
- **With DevOps/Infrastructure teams:** Implement security controls and compliance measures
- **With Project Managers:** Escalate security risks and manage remediation timelines
